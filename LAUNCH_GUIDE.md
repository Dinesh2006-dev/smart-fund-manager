# Product Launch Guide

To prepare your **Smart Fund Manager** for a production launch, follow this checklist.

## 1. Environment Configuration

### Server (`server/.env`)
Ensure your production environment variables are secure.
- **PORT**: Set to `5000` or your host's requirement.
- **JWT_SECRET**: Use a strong, random string (e.g., generated by `openssl rand -base64 32`).
- **NODE_ENV**: Set to `production`.

### Client (`client/.env`)
- **VITE_API_URL**: Point this to your live server domain for build.

## 2. Optimization

### Database
- **SQLite** is great for testing but might not scale for heavy production use. Consider migrating to **PostgreSQL** or **MySQL** if you expect thousands of concurrent users.
- **Backups**: Ensure you have a strategy to backup `server/database.sqlite` daily.

### Backend Performance
- **Compression**: Use the `compression` middleware to reduce payload sizes.
- **Security**: Use `helmet` to secure HTTP headers.

## 3. Deployment Strategy (Single Server)

The easiest way to launch is to let your **Node.js server** serve the **React frontend**.

1.  **Build the Client:**
    ```bash
    cd client
    npm run build
    ```
    This creates a `dist/` folder with optimized static files.

2.  **Update Server Logic:**
    Configure `server/index.js` to serve these static files in production mode.

## 4. Process Management

Use **PM2** to keep your application running forever, even after a crash or reboot.

1.  **Install PM2:** `npm install -g pm2`
2.  **Start App:** `pm2 start server/index.js --name "smart-fund"`
3.  **Save List:** `pm2 save`

## 5. Security Checklist
- [ ] Enforce strong passwords.
- [ ] Enable HTTPS (SSL Certificate).
- [ ] Rate Limit API requests to prevent abuse.
